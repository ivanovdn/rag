# TASK: Fix DOCX Parser — Resolve Word Auto-Numbering

## Problem

Our DOCX parser (`ingest/docx_parser.py`) **misses clause numbers** like `7.5.`, `4.1.`, `10.2.` because these numbers are auto-generated by Word's numbering engine — they're NOT in `para.text`. The parser only sees "Blogging and Social Media: Limited and occasional use..." but the rendered document shows "7.5. Blogging and Social Media: Limited and occasional use...".

This means:
- **Chunks have no clause numbers** in their metadata → citations are broken
- **Section hierarchy is lost** → "Blogging and Social Media" looks like a flat paragraph instead of section 7.5 under "Private Information"
- **Retrieval quality suffers** → a search for "section 7.5" finds nothing

## Root Cause (Diagnosed)

We ran diagnostic scripts against the actual policy DOCX files and found:

**All policy documents use Word's `numbering.xml`** to auto-generate numbers. The document has a single numbering definition (`numId=2`) with these levels:

```
ilvl=0: fmt=decimal, text='%1.'       → Section numbers: 1., 2., 3., ... 11.
ilvl=1: fmt=decimal, text='%1.%2.'    → Clause numbers:  4.1., 4.2., ... 7.5., 7.6.
ilvl=2: fmt=bullet,  text='●'         → Bullet items under clauses
ilvl=3: fmt=bullet,  text='○'         → Sub-bullet items
```

The numbers live in the XML numbering definition, not in `para.text`. Python-docx's `para.text` returns only the visible text runs, not the auto-generated numbers.

**Example from diagnostic output:**

```
IDX   RESOLVED  ILVL STYLE                TEXT
 28         4.    L0 Heading 1            Corporate Workstation and Software Use
 29       4.1.    L1 List Paragraph       Acceptable Use: All Team Members are required...
 30       4.2.    L1 List Paragraph       Exceptions: The use of personal laptops...
...
135       7.5.    L1 List Paragraph       Blogging and Social Media: Limited and occasional...
142       7.6.    L1 List Paragraph       Landline Telephones and Mobile Phones: Landline...
```

The current parser sees index 135 as just "Blogging and Social Media: Limited..." with style `List Paragraph` — it has no way to know this is clause 7.5 or that it's a subsection.

## Solution

Two changes:
1. **New module `ingest/numbering.py`** — resolves Word auto-numbering by simulating the counters from `numbering.xml`
2. **Modify `ingest/docx_parser.py`** — use the resolver to prepend numbers and determine hierarchy from `ilvl`

## Implementation

### Step 1: Create `ingest/numbering.py`

This module reads Word's `numbering.xml`, tracks counters per level, and returns the resolved number string for each paragraph.

**How Word numbering works:**
- `numbering.xml` defines abstract numbering definitions with level templates like `%1.%2.` (meaning "counter_for_level_0.counter_for_level_1.")
- Each paragraph references a `numId` and `ilvl` (indent level) in its XML properties
- When Word renders the document, it increments the counter for the current level and resets all deeper levels
- This module replicates that logic

```python
"""
Resolve auto-generated numbering from Word's numbering.xml.

Word stores numbering definitions separately from paragraph text.
When you see "7.5. Blogging and Social Media:" in the rendered document
but para.text only contains "Blogging and Social Media:", the "7.5."
comes from the numbering XML.

This module reconstructs those numbers by simulating Word's counters.

Usage:
    from ingest.numbering import NumberingResolver
    
    resolver = NumberingResolver(doc)
    
    for element in doc.element.body:
        num_info = resolver.resolve(element)
        # num_info = {"resolved": "7.5.", "ilvl": 1, "numFmt": "decimal"}
        # or None if no numbering
"""

from docx.oxml.ns import qn
from docx import Document


class NumberingResolver:
    """
    Resolves Word auto-numbering for each paragraph.
    
    Initialize once per document, then call resolve(element) for each
    paragraph element to get its resolved number string.
    """
    
    def __init__(self, doc: Document):
        self.num_to_abstract: dict[str, str] = {}
        self.abstract_levels: dict[str, dict] = {}
        self.counters: dict[str, dict[str, int]] = {}
        
        self._parse_numbering(doc)
    
    def _parse_numbering(self, doc: Document) -> None:
        """Parse numbering.xml to extract definitions."""
        try:
            numbering_part = doc.part.numbering_part
            if numbering_part is None:
                return
            numbering_xml = numbering_part._element
        except Exception:
            return
        
        # Map numId → abstractNumId
        for num_el in numbering_xml.findall(qn("w:num")):
            num_id = num_el.get(qn("w:numId"))
            abstract_ref = num_el.find(qn("w:abstractNumId"))
            if abstract_ref is not None:
                self.num_to_abstract[num_id] = abstract_ref.get(qn("w:val"))
        
        # Map abstractNumId → level definitions
        for abstract_el in numbering_xml.findall(qn("w:abstractNum")):
            abstract_id = abstract_el.get(qn("w:abstractNumId"))
            levels = {}
            for lvl_el in abstract_el.findall(qn("w:lvl")):
                ilvl = lvl_el.get(qn("w:ilvl"))
                num_fmt_el = lvl_el.find(qn("w:numFmt"))
                lvl_text_el = lvl_el.find(qn("w:lvlText"))
                start_el = lvl_el.find(qn("w:start"))
                levels[ilvl] = {
                    "numFmt": num_fmt_el.get(qn("w:val")) if num_fmt_el is not None else "decimal",
                    "lvlText": lvl_text_el.get(qn("w:val")) if lvl_text_el is not None else "",
                    "start": int(start_el.get(qn("w:val"))) if start_el is not None else 1,
                }
            self.abstract_levels[abstract_id] = levels
    
    def resolve(self, element) -> dict | None:
        """
        Resolve numbering for a paragraph element.
        
        Returns dict with:
            resolved: str  — the number string, e.g. "7.5."
            ilvl: int      — indent level (0=section, 1=clause, 2+=bullets)
            numFmt: str    — "decimal" or "bullet"
        
        Returns None if the paragraph has no numbering.
        """
        pPr = element.find(qn("w:pPr"))
        if pPr is None:
            return None
        
        numPr = pPr.find(qn("w:numPr"))
        if numPr is None:
            return None
        
        ilvl_el = numPr.find(qn("w:ilvl"))
        numId_el = numPr.find(qn("w:numId"))
        
        if ilvl_el is None or numId_el is None:
            return None
        
        ilvl = ilvl_el.get(qn("w:val"))
        num_id = numId_el.get(qn("w:val"))
        ilvl_int = int(ilvl)
        
        # Get level definitions
        abstract_id = self.num_to_abstract.get(num_id, "0")
        levels = self.abstract_levels.get(abstract_id, {})
        level_def = levels.get(ilvl, {})
        num_fmt = level_def.get("numFmt", "decimal")
        
        # Initialize counters for this numId
        if num_id not in self.counters:
            self.counters[num_id] = {}
        
        # Initialize level counter if needed
        if ilvl not in self.counters[num_id]:
            start = level_def.get("start", 1)
            self.counters[num_id][ilvl] = start
        else:
            self.counters[num_id][ilvl] += 1
        
        # Reset deeper levels
        for deeper_ilvl in list(self.counters[num_id].keys()):
            if int(deeper_ilvl) > ilvl_int:
                deeper_start = levels.get(deeper_ilvl, {}).get("start", 1)
                self.counters[num_id][deeper_ilvl] = deeper_start - 1
        
        # Build number string from lvlText template
        lvl_text = level_def.get("lvlText", "")
        resolved = lvl_text
        for lvl_idx in range(ilvl_int + 1):
            lvl_str = str(lvl_idx)
            placeholder = f"%{lvl_idx + 1}"
            if lvl_str in self.counters[num_id]:
                resolved = resolved.replace(placeholder, str(self.counters[num_id][lvl_str]))
        
        return {
            "resolved": resolved,
            "ilvl": ilvl_int,
            "numFmt": num_fmt,
        }
```

#### Edit 3: Modify the main paragraph processing loop

In the `for element in doc.element.body:` loop, **replace** the existing paragraph processing block. The current code is approximately:

```python
        if element.tag == qn("w:p"):

                continue
```

**Replace with** (the table and flush logic at the end stays unchanged):

```python
        if element.tag == qn("w:p"):
            para = para_map.get(element)
            if para is None:
                continue

            text = para.text.strip()
            if not text:
                continue

            # ── Resolve Word auto-numbering ──
            num_info = resolver.resolve(element)
            level = extract_heading_level(para)

            if num_info and num_info["numFmt"] == "decimal":
                # This paragraph has auto-generated decimal numbering
                # that is NOT in para.text. Prepend it.
                resolved = num_info["resolved"]  # e.g. "7.5."
                text = f"{resolved} {text}"

                if num_info["ilvl"] == 0:
                    # ilvl=0 + decimal = top-level section (like Heading 1)
                    # e.g. "4. Corporate Workstation and Software Use"
                    level = 1
                elif num_info["ilvl"] == 1:
                    # ilvl=1 + decimal = subsection/clause (like Heading 2)
                    # e.g. "7.5. Blogging and Social Media: ..."
                    level = 2

            if level:
                flush_all()
                idx = min(level, 3) - 1
                current_headings[idx] = text
                for i in range(idx + 1, 3):
                    current_headings[i] = ""
                current_clause_number = ""
            else:
                clause_num = extract_clause_number(text)
                if not clause_num:
                    label = extract_label(text)
                    if label and para.style.name == "List Paragraph":
                        clause_num = label
                if clause_num and current_text_buffer:
                    flush_buffer()
                    current_clause_number = clause_num
                elif clause_num:
                    current_clause_number = clause_num
                current_text_buffer.append(text)
```

**What this changes:**
- Before processing, we resolve the auto-numbering for the current paragraph
- If the paragraph has decimal numbering (not bullet), we prepend the number to the text: `"Blogging and Social Media: ..."` becomes `"7.5. Blogging and Social Media: ..."`
- We use `ilvl` to determine hierarchy: `ilvl=0` → section heading (H1), `ilvl=1` → clause heading (H2)
- Bullet items (`ilvl=2,3` with `numFmt=bullet`) pass through unchanged — they're content under clauses
- The `extract_clause_number` regex will now match the prepended numbers for any paragraphs not caught by the ilvl logic
- All existing logic for headings, flush, buffer, etc. stays exactly the same

### Step 3: Handle Edge Case — Heading 1 with numId

Some `Heading 1` paragraphs ALSO have `numId=2, ilvl=0` (like index 5, 14, 20, 28 in the diagnostic output). The current parser already detects these via `extract_heading_level(para)` because their style is "Heading 1". The new code handles this correctly:

1. `extract_heading_level` returns `1` (because style is "Heading 1")
2. `num_info` returns `{"resolved": "4.", "ilvl": 0, "numFmt": "decimal"}`
3. The decimal numbering block prepends "4." to the text AND sets `level = 1`
4. Since `level` was already 1 from the style, both paths agree

So `Heading 1` paragraphs with numbering get the number prepended AND are correctly treated as H1. No conflict.

### Step 4: Handle Documents Without Numbering

The `NumberingResolver` is safe for documents that don't use `numbering.xml`:
- `_parse_numbering()` catches exceptions and returns empty dicts
- `resolve()` returns `None` for paragraphs without `numPr` in their XML
- When `resolve()` returns `None`, the parser falls through to the existing logic
- This means documents without auto-numbering work exactly as before
